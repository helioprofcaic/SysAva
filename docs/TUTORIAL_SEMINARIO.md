# 🎓 Tutorial: Criando e Respondendo Avaliações do Tipo "Seminário" no SysAva

Este tutorial orienta professores e alunos sobre como utilizar a modalidade de avaliação "Outros" para a realização de seminários, projetos e trabalhos em grupo.

## Para Professores: Como Criar a Avaliação

1.  **Acesse o Painel Administrativo:** Vá para a aba `🛡️ Painel Administrativo`.
2.  **Gerenciar Avaliações:** Clique na aba `Gerenciar Avaliações (Provas)`.
3.  **Selecione o Contexto:** Escolha a `Turma` e a `Disciplina` onde o seminário será aplicado.
4.  **Crie a Avaliação:**
    *   No quadro "Nova Avaliação", selecione o **Tipo** `Outros`.
    *   No campo **Título**, dê um nome claro, como `Seminário: Tópicos Avançados de Programação`.
    *   Clique em `Criar Avaliação`.
5.  **Crie as Questões (Tópicos do Seminário):**
    *   Após criar a avaliação, selecione-a na lista "Avaliações Existentes".
    *   Na seção "Adicionar Questão", escolha o tipo `Subjetiva`.
    *   No **Enunciado**, descreva o tema do grupo. **É fundamental identificar o grupo no enunciado**. Exemplo: `[GRUPO 1] - Tema: O uso de ponteiros em C++`.
    *   Marque a opção **"Adicionar campo para envio de Link (GitHub/Drive)?"**. Isso é crucial para que os alunos possam enviar o material (slides, código, vídeo).
    *   Repita o processo para cada grupo/tema do seminário. Todos os alunos verão todas as questões, mas responderão apenas àquela designada ao seu grupo.

## Para Alunos: Como Responder e Entregar o Seminário

Ao acessar a avaliação do tipo "Seminário", você verá uma ou mais questões subjetivas. Seu grupo deve preparar o material (slides, documento de texto, código-fonte) e submetê-lo conforme o modelo abaixo.

### Modelo de Documento para Entrega do Seminário

Recomendamos criar um documento no Google Docs, Word (salvo como PDF) ou um arquivo `README.md` em um repositório do GitHub seguindo esta estrutura. Ao final, basta colar o link compartilhável do documento/repositório no campo "Link Externo" da sua questão.
